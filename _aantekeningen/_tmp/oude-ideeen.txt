# Private money, private code?
Public money, public code, want investering van de overheid is voor de samenleving;
open source vooral voor generieke toepassingen (nav presentatie recent), niet voor heel specifieke toepassingen (want geen hergebruik)
open source in public tech is andere open source dan die in free software
free software zijn eigen inspanningen voor eigen gebruik (tools voor de coders zelf)
maar public code is voor overheden; licentie nodig die private investeringen daarin de moeite waard maakt, om ook een economie voor innovatie bovenop die code bases te maken, anders alleen de verdienmodellen mogelijk op software "as is", dus dan moet alle innovatie vanuit de overheid komen (en die maken geen projecten omdat ze nou net niet precies ideeën hebben over waar mogelijk competitive edge zit)
